Skill này tổng hợp **lộ trình và mindset** để trở thành một BA xuất sắc — từ người mới bắt đầu đến chuyên gia giá trị cao.

### 2. How to Start a BA Career (Laura Brandenburg)

```
┌─────────────────────────────────────────────────────────────────┐
│                    BA CAREER PATHWAY                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  BƯỚC VÀO NGHỀ BA:                                              │
│  ═════════════════                                              │
│                                                                 │
│  1. IDENTIFY TRANSFERABLE SKILLS                                │
│     • Từ roles khác: QA, PM, Developer, Support, Operations    │
│     • Kỹ năng chung: Communication, Analysis, Documentation    │
│                                                                 │
│  2. BUILD YOUR BA TOOLKIT                                       │
│     □ Learn a modeling notation (BPMN, UML basics)             │
│     □ Practice writing User Stories                             │
│     □ Study BABOK fundamentals                                  │
│     □ Get hands-on with prototyping tools                       │
│                                                                 │
│  3. GET EXPERIENCE (even without the title)                     │
│     • Volunteer for BA tasks in current role                   │
│     • Propose process improvements                              │
│     • Document systems you work with                            │
│     • Ask to shadow BA colleagues                               │
│                                                                 │
│  4. TRANSITION STRATEGIES                                       │
│     • Internal move (easiest)                                  │
│     • Contract/freelance BA work                               │
│     • Junior BA positions                                      │
│     • BA roles in smaller companies (wear multiple hats)       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Seven Steps to Mastering Business Analysis (Barbara Carkenord)

```
┌─────────────────────────────────────────────────────────────────┐
│                    SEVEN STEPS TO MASTERY                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  STEP 1: UNDERSTAND THE BUSINESS NEED                           │
│  ═════════════════════════════════════                          │
│  • Nghiên cứu industry, domain                                  │
│  • Học ngôn ngữ kinh doanh của stakeholder                     │
│  • Xác định "gap" giữa hiện tại và mong muốn                   │
│                                                                 │
│  STEP 2: PLAN YOUR APPROACH                                     │
│  ══════════════════════════                                     │
│  • Chọn kỹ thuật phù hợp với context                           │
│  • Lên kế hoạch elicitation                                    │
│  • Xác định deliverables                                        │
│                                                                 │
│  STEP 3: ELICIT REQUIREMENTS                                    │
│  ════════════════════════════                                   │
│  • Interviews, Workshops, Observation                          │
│  • Document Analysis, Prototyping                              │
│  • Validate và confirm hiểu đúng                               │
│                                                                 │
│  STEP 4: ANALYZE REQUIREMENTS                                   │
│  ═════════════════════════════                                  │
│  • Organize, Prioritize, Model                                 │
│  • Identify gaps và conflicts                                  │
│  • Verify quality attributes (INVEST, etc.)                    │
│                                                                 │
│  STEP 5: COMMUNICATE REQUIREMENTS                               │
│  ═════════════════════════════════                              │
│  • Tailor format cho audience                                  │
│  • Use visuals effectively                                     │
│  • Present và defend requirements                              │
│                                                                 │
│  STEP 6: SUPPORT SOLUTION IMPLEMENTATION                        │
│  ═══════════════════════════════════════                        │
│  • Clarify questions during development                        │
│  • Manage scope changes                                        │
│  • Participate in UAT                                          │
│                                                                 │
│  STEP 7: EVALUATE THE SOLUTION                                  │
│  ══════════════════════════════                                 │
│  • Measure against success criteria                            │
│  • Document lessons learned                                    │
│  • Propose improvements                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
