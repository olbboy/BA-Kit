## 🎯 MỤC ĐÍCH

Skill này tổng hợp **kỹ năng mềm và năng lực lãnh đạo** — yếu tố quyết định sự thành công của một BA. Kỹ thuật giỏi mà không thể thuyết phục stakeholder thì vô nghĩa.

## 🧠 KIẾN THỨC CỐT LÕI

### 1. Start With Why (Simon Sinek) — Bắt Đầu Từ "Tại Sao"

```
┌─────────────────────────────────────────────────────────────────┐
│                    THE GOLDEN CIRCLE                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                      ┌───────────┐                              │
│                      │    WHY    │   ← Niềm tin, Mục đích       │
│                      │ (Purpose) │   "Chúng ta tồn tại để...?"  │
│                      └─────┬─────┘                              │
│                      ┌─────┴─────┐                              │
│                      │    HOW    │   ← Cách làm khác biệt       │
│                      │ (Process) │   "Điều gì khiến ta khác?"   │
│                      └─────┬─────┘                              │
│                      ┌─────┴─────┐                              │
│                      │   WHAT    │   ← Sản phẩm, Kết quả        │
│                      │ (Product) │   "Chúng ta làm gì?"         │
│                      └───────────┘                              │
│                                                                 │
│  INSIGHT CHÍNH:                                                 │
│  ═══════════════                                                │
│  "People don't buy WHAT you do, they buy WHY you do it."        │
│                                                                 │
│  ÁP DỤNG CHO BA:                                                │
│  ═══════════════                                                │
│  • Khi trình bày requirement, bắt đầu bằng WHY (Business Value) │
│  • Stakeholder không quan tâm spec chi tiết — họ quan tâm       │
│    "Điều này giúp gì cho tôi?"                                  │
│  • Mọi tính năng phải trace về một Business Objective           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2. How to Win Friends and Influence People (Dale Carnegie)

```
┌─────────────────────────────────────────────────────────────────┐
│                    CARNEGIE'S PRINCIPLES                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  PHẦN 1: XÂY DỰNG MỐI QUAN HỆ                                   │
│  ═════════════════════════════                                  │
│  1. Không chỉ trích, lên án, hay phàn nàn                       │
│  2. Chân thành khen ngợi                                        │
│  3. Khơi gợi khát vọng của người khác                           │
│                                                                 │
│  PHẦN 2: KHIẾN NGƯỜI KHÁC YÊU QUÝ BẠN                           │
│  ═════════════════════════════════════                          │
│  4. Thực sự quan tâm đến người khác                             │
│  5. Mỉm cười                                                    │
│  6. Nhớ tên người khác (Tên là âm thanh ngọt ngào nhất)         │
│  7. Lắng nghe tốt, khuyến khích người khác nói về họ            │
│  8. Nói về sở thích của người khác                              │
│  9. Làm cho người khác cảm thấy họ quan trọng                   │
│                                                                 │
│  PHẦN 3: THAY ĐỔI SUY NGHĨ NGƯỜI KHÁC                           │
│  ═══════════════════════════════════                            │
│  10. Tránh tranh cãi                                            │
│  11. Tôn trọng quan điểm đối phương                             │
│  12. Nếu sai, thừa nhận nhanh chóng                             │
│  13. Bắt đầu một cách thân thiện                                │
│  14. Đặt câu hỏi để người khác tự đi đến kết luận               │
│  15. Để người khác nói nhiều hơn                                │
│  16. Để người khác cảm thấy ý tưởng là của họ                   │
│  17. Nhìn từ góc độ của người khác                              │
│  18. Đồng cảm với mong muốn của họ                              │
│  19. Kêu gọi động cơ cao quý hơn                                │
│  20. Dramatize your ideas (Làm ý tưởng sống động)               │
│  21. Thử thách họ                                               │
│                                                                 │
│  ÁP DỤNG CHO BA:                                                │
│  ═══════════════                                                │
│  • Trong conflict resolution: Nguyên tắc 10-11-12               │
│  • Trong elicitation: Nguyên tắc 7-8 (Lắng nghe, hỏi về họ)     │
│  • Trong stakeholder buy-in: Nguyên tắc 16-17 (Góc nhìn họ)     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3. Business Analysis and Leadership (Penny Pullan) — BA as Change Agent

```
┌─────────────────────────────────────────────────────────────────┐
│                    BA AS CHANGE AGENT                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  THE BA'S LEADERSHIP CHALLENGE:                                 │
│  ════════════════════════════════                               │
│  • BA thường không có quyền lực chính thức (formal authority)   │
│  • Phải dẫn dắt thông qua ẢNH HƯỞNG (influence)                 │
│  • Đối mặt với kháng cự thay đổi (resistance to change)         │
│                                                                 │
│  PULLAN'S INFLUENCE MODEL:                                      │
│  ═════════════════════════                                      │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                                                           │  │
│  │  CREDIBILITY + RELATIONSHIP + LOGIC = INFLUENCE           │  │
│  │                                                           │  │
│  │  • Credibility: Chuyên môn, Track record                  │  │
│  │  • Relationship: Trust, Rapport, Empathy                  │  │
│  │  • Logic: Facts, Data, Clear reasoning                    │  │
│  │                                                           │  │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  DEALING WITH RESISTANCE:                                       │
│  ═════════════════════════                                      │
│  1. Acknowledge emotions first (Đừng phản bác ngay)            │
│  2. Seek to understand (Tại sao họ kháng cự?)                  │
│  3. Involve them in the solution (Họ là một phần giải pháp)    │
│  4. Communicate benefits in their terms (Lợi ích cho HỌLÀ gì?) │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4. Making Workshops Work (Penny Pullan) — Nghệ Thuật Facilitation

```
┌─────────────────────────────────────────────────────────────────┐
│                    WORKSHOP FACILITATION                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  BEFORE THE WORKSHOP:                                           │
│  ════════════════════                                           │
│  □ Define clear objectives (Một workshop = Một mục tiêu chính)  │
│  □ Identify the right participants                              │
│  □ Choose appropriate techniques                                │
│  □ Prepare materials and environment                            │
│  □ Send pre-work if needed                                      │
│                                                                 │
│  DURING THE WORKSHOP:                                           │
│  ═══════════════════                                            │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │ OPEN (10%)       │ Welcome, Agenda, Ground Rules        │   │
│  ├───────────────────┼─────────────────────────────────────┤   │
│  │ DIVERGE (30%)    │ Generate ideas (Brainstorm, etc.)    │   │
│  ├───────────────────┼─────────────────────────────────────┤   │
│  │ EXPLORE (30%)    │ Discuss, Analyze, Cluster            │   │
│  ├───────────────────┼─────────────────────────────────────┤   │
│  │ CONVERGE (20%)   │ Prioritize, Decide, Agree            │   │
│  ├───────────────────┼─────────────────────────────────────┤   │
│  │ CLOSE (10%)      │ Summarize, Next Steps, Thank         │   │
│  └───────────────────┴─────────────────────────────────────┘   │
│                                                                 │
│  FACILITATION TECHNIQUES:                                       │
│  ═════════════════════════                                      │
│  • Silent Brainstorming (Sticky notes first)                   │
│  • Round Robin (Everyone speaks in turn)                       │
│  • Dot Voting (Quick prioritization)                           │
│  • Parking Lot (Capture off-topic items)                       │
│  • Fishbowl (Inner/outer circles)                              │
│                                                                 │
│  AFTER THE WORKSHOP:                                            │
│  ═══════════════════                                            │
│  □ Send summary within 24 hours                                 │
│  □ Clarify action items and owners                              │
│  □ Follow up on commitments                                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 📋 LEADERSHIP FRAMEWORK CHO BA

| Tình huống | Kỹ thuật áp dụng |
|------------|------------------|
| Trình bày requirement | Bắt đầu với WHY (Sinek) |
| Stakeholder kháng cự | Acknowledge → Understand → Involve (Pullan) |
| Xung đột ý kiến | Nguyên tắc 10-11-12 (Carnegie: Tránh tranh cãi) |
| Workshop planning | OPEN → DIVERGE → EXPLORE → CONVERGE → CLOSE |
| Xây dựng trust | Nguyên tắc 4-7-9 (Carnegie: Quan tâm, Lắng nghe, Tôn trọng) |
| Thuyết phục Executive | Credibility + Relationship + Logic (Pullan) |
